<MyH2>What is Routing?</MyH2>
Routing is the process of figuring out what resource to return when a given url is visited. This process is normally done on the server.
The piece of code that carries out this process is called a Router.

<Info>
    Frameworks like React and Vue handle routing on the browser (client-side)
</Info>

### How does Routing work?

While different Frameworks handle Routing a little bit differently, the basic pattern remains the same.
- You visit a url
- The Router checks the url and other details about the request to know the right resource to return to the client (for example your browser)
- If all parameters including the url are valid it returns the appropriate resource or an appropriate error message (for example a "not found" page)

#### Let's see an example

Let's say you want to visit your favourite website. You might visit the url below 
<TerminalBLock text="localhost:3000/myfavouritesite.com/"/>

If your favourite website is a one where you can stream music, you might want to see all the trending songs of the week.
To do that, you visit the url
<TerminalBLock text="localhost:3000/myfavouritesite.com/trending"/>

In this case, the router scans the url, it sees the url is valid, then it returns the page where you see all the trending songs for the week.

While trying to visit the page to see the trending song, you might end up typing the wrong url, maybe you spelled "trending" but without the "n" as in the url below
<TerminalBLock text="localhost:3000/myfavouritesite.com/treding"/>

Now, the router checks the url, it discovers there's no such Url in its registry so it returns a not-found page.
That's it, a very basic description of how routing works!.

<Info>
    The "resource" returned after a request is confirmed valid could be an HTML page like in the example above, a downloadable file
    audio, video, JSON, or even XML and many more formats.
</Info>

<MyH2>How to create a route</MyH2>

To create a new route, simply add a new folder inside the app folder of your Next.js project. The name of the folder should be the name of the route you want to create. For example, if you want to create a '/service' route, the name of the folder should be <em>Service</em>. If you want to create a '/projects' route, the name of the folder should be <em>projects</em>.
After creating the folder, create a new file inside the folder you just created. That file should
be named page.tsx. The image below would make it clearer. 

<SampleImage src={sample_image} alt="services route created in a Next.js project"/>

<Info>
    - In the image above, the project uses the <em>src</em> directory. it's not a requirement, you can use the app router 
    without using the src directory in your project.
    - Page.tsx (Page.jsx if you're not using TypeScript ) is a special file in Next.js. When the server sees this file in a directory, it renders the directory as a route.
</Info>

Please notice that in the image above, the name of the folder inside the app folder is <em>Services</em>.
Your folder name could be anything you want. it all depends on how you want your routes to be named. Inside the service folder 
we have a file named "page", a tsx file. 

The page.tsx file is going to hold the content to be displayed when people visit the route. In this instance, the page.tsx file holds the 
content to be displayed when people visit the "/service" route.
